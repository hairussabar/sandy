The SingletonError class is used as a workaround for private constructors not existing in ActionScript 3.0. Every singleton class in Sandy has a private static variable called instance. The instance variable is given a reference to an instance of the class the first time the class constructor or getInstance() is called. If an attempt is made to instantiate the class a the second time, a SingletonError will be thrown. Always use the static method Class.getInstance() to get an instance of the class.
